# Data Model: 003 — Performance Optimization

Спека 003 не вводит новых сущностей, полей или хранилищ.

**Изменения только в реализации существующих контрактов:**

- **Сериализация JSON**: те же структуры данных (логи, replay, конфиг), формат вывода сохраняется (orjson с OPT_SORT_KEYS). Типы данных не меняются.
- **Хеши**: те же поля (identity_hash, configuration_hash, memory_signature, transit_signature, replay signature), другой алгоритм (xxhash вместо hashlib). Значения хешей меняются — breaking change, контракт 002+.
- **Циклы**: те же входы/выходы (параметры, оси, агрегаты); меняется только способ вычисления (векторизация или оптимизированные циклы).

Отдельный data-model.md для 002 и существующих модулей остаётся актуальным. Данный файл фиксирует отсутствие новых сущностей для 003.
